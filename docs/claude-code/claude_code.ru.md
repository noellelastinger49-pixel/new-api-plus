# Руководство по настройке канала Claude Code

## Обзор

Claude Code — это официальный инструмент командной строки от Anthropic, поддерживающий авторизацию доступа к API Claude через OAuth. New API поддерживает Claude Code как самостоятельный тип канала (Type 200), позволяя повторно использовать учётные данные CLI Claude Code.

## Возможности

- **Аутентификация OAuth**: Авторизация через поток OAuth 2.0 от Anthropic
- **Автоматическое обновление**: Автоматическое обновление истёкших access-токенов
- **Совместимость форматов учётных данных**: Поддержка нативного формата CLI Claude Code и внутреннего формата
- **Веб-интерфейс авторизации**: Графический процесс авторизации без ручного копирования токенов

## Настройка канала

### 1. Создание канала Claude Code

1. Перейдите в панель администратора → Управление каналами → Добавить канал
2. Выберите тип канала: **Claude Code**
3. Укажите название канала (например: `Claude Code - Personal`)
4. Настройте ключ (два способа)

![1.new_channel.png](1.new_channel.png)

### 2. Способы настройки учётных данных

#### Способ 1: Веб-авторизация (Рекомендуется)

1. Нажмите кнопку «Авторизовать»
2. В открывшемся окне войдите в аккаунт Anthropic
3. Подтвердите авторизацию доступа Claude Code
4. После успешной авторизации учётные данные будут заполнены автоматически

![2.gen_credentials.png](2.gen_credentials.png)

#### Способ 2: Ручная вставка учётных данных

Если у вас уже есть учётные данные CLI Claude Code, вы можете вставить их напрямую в формате JSON.

**Формат CLI Claude Code** (`~/.claude/config.json`):

```json
{
  "claudeAiOauth": {
    "accessToken": "sk-ant-oat01-...",
    "refreshToken": "sk-ant-ort01-...",
    "expiresAt": 1782662007674,
    "scopes": ["user:file_upload", "user:inference", "user:mcp_servers", "user:profile", "user:sessions:claude_code"],
    "subscriptionType": "pro",
    "rateLimitTier": "default_claude_ai"
  }
}
```

### 4. Управление учётными данными

### Механизм автоматического обновления

New API автоматически обновляет учётные данные Claude Code в следующих случаях:

1. **Проверка перед запросом**: Проверка истечения срока действия токена перед каждым API-запросом (за 5 минут до истечения)
2. **Плановая задача**: Система регулярно сканирует все каналы Claude Code для обновления учётных данных, срок действия которых скоро истекает
3. **Повторная попытка при ошибке**: При получении ответа 401 Unauthorized автоматическая попытка обновления и повторный запрос

### Ручное обновление

На странице списка каналов каналы Claude Code отображают кнопку «Обновить учётные данные» для ручного запуска обновления.

### Просмотр учётных данных

Суперадминистраторы могут просматривать полную информацию об учётных данных (включая refresh token) на странице редактирования канала.

## Часто задаваемые вопросы

### В: Что делать, если учётные данные истекли?

О: New API автоматически обновляет учётные данные. В случае сбоя автоматического обновления:
1. Нажмите кнопку «Обновить учётные данные» для ручного обновления
2. Повторите авторизацию (нажмите кнопку «Авторизовать»)

### В: Какие типы подписки поддерживаются?

О: Поддерживаются все типы подписки Anthropic:
- Free (бесплатная версия)
- Pro (профессиональная версия)
- Team (командная версия, доступ через учётные данные Pro)

Доступные модели зависят от вашего типа подписки.

### В: Как получить учётные данные CLI Claude Code?

О: Установите и войдите в CLI Claude Code:

```bash
# macOS
brew install claude

# Вход
claude auth login
```

После входа учётные данные сохраняются в `~/.claude/config.json`. На macOS они находятся в «Связке ключей».

### В: Как предотвратить блокировку Claude Code?
1. Если у вас только один аккаунт: используйте его только на сервере, запретите использование тех же учётных данных с других IP-адресов;
2. Если у вас несколько аккаунтов: используйте чистый ISP-сервер в качестве прокси, и получайте учётные данные каждого аккаунта только на этом чистом ISP-сервере;

## Технические детали

### Поток авторизации OAuth

1. Пользователь нажимает кнопку «Авторизовать»
2. Фронтенд открывает окно авторизации OAuth
3. Пользователь входит и авторизуется в Anthropic
4. Anthropic перенаправляет на бэкенд New API `/api/oauth/claude-code/callback`
5. Бэкенд обменивает код авторизации на access token и refresh token
6. Фронтенд получает полные учётные данные и заполняет форму

### Поток обновления токенов

1. Обнаружение истечения или скорого истечения срока действия токена
2. Использование refresh token для вызова конечной точки OAuth `/oauth/token` Anthropic
3. Получение нового access token и refresh token
4. Обновление ключа канала в базе данных
5. Повторная попытка исходного запроса с новым токеном

### Формат хранения учётных данных

Поле `key` в базе данных хранится в формате JSON:

```json
{
  "access_token": "sk-ant-oat01-...",
  "refresh_token": "sk-ant-ort01-...",
  "expires_at": 1782662007674
}
```

Обёрточный формат `claudeAiOauth`, отправленный фронтендом, автоматически разворачивается и нормализуется на бэкенде.

## Связанные интерфейсы

### Точка входа авторизации OAuth

```
GET /api/oauth/claude-code/login
```

Возвращает URL авторизации OAuth Anthropic.

### Обратный вызов OAuth

```
GET /api/oauth/claude-code/callback?code=xxx&state=xxx
```

Принимает обратный вызов OAuth от Anthropic и обменивает токен.

### Ручное обновление учётных данных

```
POST /api/channel/:id/refresh-credential
```

Запускает обновление учётных данных для конкретного канала.
